# 30-Minute Deployment Checklist

This guide is optimized for shipping quickly.

## 1) API keys and where to paste

### A. Supabase Project Settings
Get these from Supabase dashboard > Project Settings > API.

- `Project URL` -> use for:
  - `web/.env` as `NEXT_PUBLIC_SUPABASE_URL`
  - backend `.env` as `SUPABASE_URL`
- `anon public key` -> use for:
  - `web/.env` as `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- `service_role key` -> use for:
  - backend `.env` as `SUPABASE_KEY`

Note: Do NOT put service role key in frontend env.

### B. Google Safe Browsing API key
Get from Google Cloud Console > APIs & Services > Credentials.

- backend `.env` -> `GOOGLE_SAFE_BROWSING_API_KEY`

### C. Redis URL
Use Upstash/Redis Cloud or your host Redis URL.

- backend `.env` -> `REDIS_URL`

## 2) Create env files

### Backend env (project root `.env`)
Copy from `.env.example` and fill:

```env
SUPABASE_URL=
SUPABASE_KEY=
REDIS_URL=
GOOGLE_SAFE_BROWSING_API_KEY=
```

### Frontend env (`web/.env.local` for local OR Vercel env vars)
Copy from `web/.env.example` and fill:

```env
NEXT_PUBLIC_SITE_URL=
NEXT_PUBLIC_SUPABASE_URL=
NEXT_PUBLIC_SUPABASE_ANON_KEY=
ENFORCE_DOMAIN_RESTRICTION=true
ALLOWED_EMAIL_DOMAIN=@vitbhopal.ac.in
NEXT_PUBLIC_DEV_MODE=false
```

Recommended quick demo toggles:
- Strict college-only demo: `ENFORCE_DOMAIN_RESTRICTION=true`
- Open testing mode: `ENFORCE_DOMAIN_RESTRICTION=false`

## 3) Supabase auth configuration

In Supabase > Authentication > URL Configuration:

- Site URL:
  - Local: `http://localhost:3000`
  - Production: your deployed web URL

- Additional Redirect URLs:
  - Local callback: `http://localhost:3000/auth/callback`
  - Production callback: `https://<your-domain>/auth/callback`

In Supabase > Authentication > Providers > Google:
- Enable Google provider
- Add Google OAuth Client ID and Secret

## 4) Database quick setup

Create table `study_materials` if missing:

```sql
create table if not exists public.study_materials (
  id bigint generated by default as identity primary key,
  subject text not null unique,
  flashcards jsonb not null default '[]'::jsonb,
  quiz_questions jsonb not null default '[]'::jsonb,
  created_at timestamptz not null default now()
);
```

## 5) Deploy targets

### Frontend (Vercel)
- Root directory: `web`
- Framework: Next.js
- Add all variables from section 2 frontend env

### Backend (Render/Railway/Fly)
- Start command:
  - `uvicorn app.main:app --host 0.0.0.0 --port $PORT`
- Add all variables from section 2 backend env
- Ensure Redis is reachable from backend host

## 6) Smoke test after deploy

1. Open landing page.
2. Sign in with Google.
3. If strict mode is on, non-`@vitbhopal.ac.in` account should be rejected.
4. Enter study room and allow camera/mic.
5. Backend health checks:
   - `GET /`
   - `GET /api/v1/auth/health`
   - `GET /api/v1/matching/health`
   - `GET /api/v1/study-materials/health`

## 7) Known MVP limitations (acceptable for hackathon demo)

- Study room currently uses local media simulation (full peer wiring not yet integrated in page).
- Report/block and full moderation pipelines are not complete.
- Multi-party room support is not yet production-ready.

For judges, demo the verified login, matching backend readiness, study kit generation/cache, and safety intent clearly.
